Create a paragraph and unordered list for the single-table SELECT statements. Include a separate list item for the SQL used to generate each statement in the activity. The first item / statement would be SELECT * FROM CUSTOMERS;

  1. Select all customers (SELECT *).
    2.  SELECT * FROM Customers;
  2. Select only customer id and customer name fields (SELECT fields).
    3.  SELECT CustomerID, CustomerName FROM Customers;
  3. Select customer name, address, city, and postal code for all customers from the United Kingdom (WHERE field = 'value').
    4.  SELECT CustomerName, Address, City, PostalCode FROM Customers WHERE Country = 'United Kingdom';
  4. Select contact name and customer name for all customers, sorted alphabetically by contact name (ORDER BY field).
    5.  SELECT ContactName, CustomerName FROM Customers ORDER BY ContactName;
  5. Count the total number of customers (COUNT(*)).
    6.  SELECT COUNT(*) FROM Customers;
  6. Count the number of customers from each country (GROUP BY).
    7.  SELECT Country, COUNT(*) FROM Customers GROUP BY Country;
  7. Count the number of customers from each country and sort the list in descending order by count and ascending order by country in case of a tie (GROUP BY, ORDER BY).
    8.  SELECT Country, COUNT(*) FROM Customers GROUP BY Country ORDER BY COUNT(*) DESC, Country ASC;
  8. Count the number of customers from each country and sort the list in descending order by count and ascending order by country in case of a tie, listing only those countries with more than 10 customers (GROUP BY, HAVING, ORDER BY).
    9.  SELECT Country, COUNT(*) FROM Customers GROUP BY Country HAVING COUNT(*) > 10 ORDER BY COUNT(*) DESC, Country ASC;
